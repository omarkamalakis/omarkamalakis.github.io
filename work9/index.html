Homelab Part 1: Initial Plans

Gabe Kamalakis published on January 02, 2020
3 min, 458 words

After procuring a used workstation, I set off to build a homelab and take my first steps into virtualization...

The machine:

The computer is a used Dell Precision workstation that a church did not need anymore. While being fairly light on storage options, the PC has a Xeon socket which makes it perfect for a server.

Specifications

  • CPU -> Dual Core Xeon (will upgrade to hexacore Xeon soon)
  • RAM -> 8 GB
  • Storage -> 500gb with 3 total drive slots (will add 4TB drive from Raspberrypi)
  • Networking -> standard Dell motherboard NIC (unknown transfer rates)
  • Video Card -> Unknown

Software Plans

This lab will will act as a bare-metal (Type 1) hypervisor for around 4-5 virtual machines (with 3-4 running 24/7). The hypervisor will either be ProxMox or using a minimal Fedora installation using KVM. The Computer has support for hardware virtualization. Each VM will be using a very lightweight Linux distribution called Alpine Linux and run the bare minimum for the software that it needs to run. Here are some tasks that the homelab will be running:

  • Plex Media Server
  • Minecraft Server
  • A web server (for this website)
  • An email server
  • NextCloud
  • Synapse
  • PfSense
  • TaskServer
  • Remote persistent TMUX
  • DevOps:
    • Ansible
    • Git repositories
    • Kubernetes
  • A documentation hub (like a wiki)
  • System images for laptop/PC
  • Filesystem backups
  • Personal AUR
  • Spacewalk

Many of these are experimental, so if they don't work or I need the processing power for other things (Minecraft/Plex), then I can easily suspend/freeze these VMs.

Checking the Host:

To actually begin working on the homelab, I began by actually checking if the hardware that we got actually works. It had come with an (presumably) empty hard drive and working video cards along with 4 GB of RAM. I found a few cables, including a DP -> DVI converter since it had no HDMI ports (and I have no Display Port cables). I also cracked open the case to see how much dust had built up within the case. The case itself is very modular, since it was built as a workstation that had to be quick to repair as well as a bit space efficient. The case was extremely dusty and had to be cleaned out carefully. I also tore off the CPU heatsink and repasted it since the thermal paste was completely dry. After servicing the PC turned on and posted without too many problems. I made a few tweaks in the bios and ran a diagnostics to make sure everything was working properly. I ran into my first problem, which is that it does not like booting without all 3 of the HDD slots populated (it thinks they are missing and require a key press to boot). This will become a problem later.
